# Source: Docs

Framework documentation, API references, official guides — things you'll want to refer back to verbatim.

## Naming

`{framework-or-product}-{topic}.md` — e.g. `apns-push-notifications.md`, `kubernetes-deployments.md`

## What goes here

- Excerpts from official docs (Apple Developer, Google Cloud, framework READMEs)
- Pasted CLI help output worth keeping
- Schema definitions, type declarations, protocol specs

What does NOT go here: your interpretation or synthesis. That belongs in `research/topics/`.
